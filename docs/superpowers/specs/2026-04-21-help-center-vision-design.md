# Longbridge Help Center — AI 原生帮助中心产品设计规格

**目标：** 将 Longbridge Docs 从"静态操作手册"升级为"AI 原生帮助中心"——会回答、会指路、会演示、会根据用户账户状态说话。

**架构：** 五层堆叠：AI 层（对话/总结/指引/对比）→ 内容层（结构化知识 + 富媒体 + 交互 widget）→ 关联层（语义图谱 + 任务串联）→ 入口层（搜索/对话/App 深链/邮件回流）→ 多端层（H5/Pad/Web 差异化形态）。

**技术栈：** VitePress 1.6, Vue 3.5, TypeScript, UnoCSS；AI 层供应商待定，通过能力契约接口隔离。

---

## 1. 产品定位

| 维度 | 当前状态 | 目标状态 |
|---|---|---|
| 角色 | 文档站（Docs） | 帮助助手（Help + Assistant） |
| 用户动作 | 翻目录、读文章 | 问问题、看演示、一步做完 |
| 内容形态 | 静态 Markdown | 富媒体 + 结构化 + 交互式 |
| 入口 | 首页导航 | 对话框 / App 内跳转 / 搜索直达 |
| 智能程度 | 无 AI | 检索增强问答、AI 摘要、AI 指引、AI 对比 |
| 平台策略 | Web 单端响应式 | H5 / Pad / Web 三形态差异化 |
| 度量指标 | PV | 问题自助解决率、任务完成率、客服工单减量 |

## 2. 目标用户与核心任务（JTBD）

### 用户分层

- **潜在新客**：注册前了解"能不能用""要多少钱""怎么开户"

## 3. 核心产品能力

### 3.1 AI 层（核心差异化）

#### Ask Longbridge（对话式帮助）
- 全站常驻悬浮入口（`⌘K` 桌面端，H5 浮标）
- 自然语言输入："入金多久到账？""港股交易时间""美股分红税怎么申报"
- 回答格式固定：`一句话结论` + `关键数据/参数` + `操作步骤`（可跳 App）+ `引用来源`（可点原文）
- 引用强制可追：每条回答底部必带 N 条段落级引用卡
- 拒答策略：税务建议/投资建议/客户账户操作类统一回落人工话术
- 多语言：中文/英文/粤语

**P0 实现约束（检索增强，非自由生成）：**
- 语义搜索（向量 + 关键字）命中原文段落 → LLM 只做"归纳 + 引用"，不自由发挥
- 所有输出必须有引用 id，引用 id 必须属于本次检索集合，否则整条丢弃

#### Smart Summary（文章 AI 摘要）
- 每篇文章顶部：`3 条要点 + 1 个关键数字 + 1 个常见坑`
- P0：离线批量生成 + 人工抽检，不实时调用 LLM
- P1：登录用户可触发"按我的账户类型生成个性化摘要"

#### AI 导游 / Playbook（P1）
- 选定任务（如"完成首笔港股交易"），AI 拆成 5-7 步
- 每步含：文字说明 + App 截图/短视频 + 一键深链 + "我卡在这一步"召唤客服（附上下文）
- 三条优先任务：开户+首笔交易、入金、税务申报

#### Smart Compare（P1）
- 用户选两个对象（港股账户 vs 美股账户、市价单 vs 限价单）
- AI 生成对比表 + 场景化建议
- 对比源来自结构化知识，不是让 AI 自由发挥

#### What's Changed（P2）
- 监听文档变更 + 产品公告
- 用户订阅领域有变化时，自动生成"变了什么/对你的影响/要不要做什么"
- 邮件/App 推送回流帮助中心

#### Translator to Plain（P2）
- 用户选中"熊证""引伸波幅""期权 Delta"→ 出现"讲人话"气泡
- 底层：术语库 + LLM 重写

#### 账户感知（P1，登录后启用）
- 知道用户的账户类型、已开通产品
- 回答自动变体：不推荐用户没开通的产品、不展示不相关地区文档
- 合规红线：只读不写，数据不出站，敏感字段脱敏，用户明确授权

### 3.2 内容层（超越 Markdown）

#### 已实现的 P0 组件

| 组件 | 用途 |
|---|---|
| `<Callout type>` | 提示框（info/tip/warning/danger）|
| `<Steps>` + `<Step title>` | 有序步骤指引，自动编号 |
| `<Video src poster caption>` | 16:9 视频嵌入 |
| `<Screenshot src spots caption>` | 截图 + 热区标注 + hover 说明 |
| `<RateTable dimensions rows keys>` | 多维度费率表，Tab 切换 |
| `<Calculator>` | 港股交易费用实时计算器 |
| `<AppDeepLink href label qrFallback>` | App 深链按钮，Web 降级 QR 码 |
| `<CategoryLanding>` | 分类落地页模板（图标/标题/文章列表/相关链接）|
| `<ArticleHeader>` | 文章顶部：适用市场标签 + 最后审校日期 + 过期警告 |

#### 待实现的内容组件（P1）

- `<FlowDiagram>`：流程图，节点可点击跳对应文档
- `<FAQ>` / `<Accordion>`：长问答折叠
- `<Timeline>`：时间轴（入金到账、新股流程）
- `<Compare>`：两列/三列对比卡
- `<Embed>`：嵌入 Figma / Loom / YouTube / 内部视频 CDN
- `<Spec>`：字段定义表（产品参数、合规字段）

#### 结构化知识源（P1）
- 同一份"港股交易费用"同时以 Markdown 文章和 JSON schema 存在
- AI / Calculator / Compare 都从结构化源读，文章只是"人类可读视图"
- Frontmatter 约定：`appliesTo`（市场）、`lastVerifiedAt`、`entities`（知识图谱节点）、`playbook`（所属任务串）、`sources`（监管引用）

#### 语义关联（P1）
- 实体识别 + hover 微卡片（定义 + 相关文章 + 讲人话）
- 侧栏展示"相关规则、费用、错误处理"
- 跨文档任务串联（Playbook 进度条）

### 3.3 入口层

| 入口 | 当前 | 目标 |
|---|---|---|
| 直接访问 | ✅ | 保持 |
| App 内 WebView | 无上下文 | App 任何页面"?"按钮直达对应帮助上下文 |
| 订单失败/拒单 | 无 | 带错误码直达解决方案 |
| 邮件/站内信 | 无 | 带 UTM + context，帮助中心识别并回应 |
| 客服工单 | 独立系统 | AI 回答不了时"升级客服"，工单自动附加用户看过的文档/问过的问题 |
| SEO | 基础 | FAQ schema、HowTo schema，让 Google/Perplexity 抓取 |

### 3.4 多端形态（差异化，非响应式）

| 端 | 主场景 | 首屏策略 | 核心交互 |
|---|---|---|---|
| **H5（App WebView 内）** | 正在操作，卡住了 | AI 提问框 + 当前 App 页面对应帮助 + Smart Summary | 单手滑动、卡片全宽、关闭=回 App |
| **H5（独立浏览器）** | 路上看、别人发的链接 | 文章内容优先 + 底部常驻提问栏 + 分享卡 | 可复制链接、分享、轻量目录 |
| **Pad** | 坐客户旁展示 | 两栏：左目录 + 右内容 + **演示模式** | 演示模式：字号加大、投屏友好 |
| **Web 桌面** | 深度查阅、客服研究 | 三栏：导航 + 内容 + AI 助手 + 结构化数据 | 键盘导航、多 Tab 对比、`⌘K` |

## 4. 首页信息架构

```
┌── 顶部状态区（感知用户）────────────────────────────────────┐
│ · 已登录：你好，Li。账户 HK·US，上周你提过 3 个问题          │
│ · 未登录：选择你的场景（新手 / 已开户 / 专业用户）           │
│ · 带 context（从 App 跳入）：关于"港股下单失败"，我来帮你    │
└────────────────────────────────────────────────────────────┘

┌── Ask Longbridge（首屏主角）──────────────────────────────┐
│   💬 问我任何问题                                         │
│   热门：港股交易时间 · 入金到账 · 美股税务 · 期权开通        │
│   [🎤 语音] [📎 贴错误码/截图]                              │
└────────────────────────────────────────────────────────────┘

┌── 我正在做（任务化）────────────────────────────────────────┐
│ · 开户  入金  交易  打新股  报税  出金                       │
│ 点某个 → 进 Playbook（任务串联）                           │
└────────────────────────────────────────────────────────────┘

┌── 为你挑选（个性化，P1）───────────────────────────────────┐
│ · 你上次看到"港股 IPO 认购流程"第 3 步，继续？              │
│ · 你订阅的"期权规则"昨日更新                               │
│ · 你账户未开通：A 股通（了解条件）                          │
└────────────────────────────────────────────────────────────┘

┌── 深入了解（16 领域，加权视觉层级）────────────────────────┐
│ Primary（大卡）：新手入门 / 开户与账户 / 入金 / 股票交易    │
│ Secondary（中卡）：出金 / 资金划转 / 衍生品 / 新股 / 融资 / 基金 │
│ Tertiary（pill）：行情数据 / 资产账单 / 活动奖励 / 合规税务 / … │
└────────────────────────────────────────────────────────────┘
```

**反模板要点：**
- 主角是**提问框**，不是搜索框（搜索是提问的降级）
- "你正在做什么"（任务）高于"你属于哪个分类"（目录）
- 16 领域按重要性和访问热度加权，不是等大卡片


## 6. AI 能力接口契约

所有 AI 能力以接口形式定义，供应商待定（Anthropic / OpenAI / 自研皆可热插拔）。

| 能力 | 输入 | 输出 | 用于 |
|---|---|---|---|
| `search(q, filters)` | 自然语言 + 过滤条件 | 排序后的段落 + 文档引用 | Ask v0.1 主干、站内搜索 |
| `ask(q, context, citations)` | 问题 + 检索段落 | 归纳答案（限定格式）+ 必带引用 id | Ask v0.1 回答合成 |
| `summarize(doc)` | 文档内容 | 3 要点 + 1 关键数字 + 1 常见坑 | Smart Summary |
| `classify(q)` | 问题 | 路由标签（可答 / 合规拒答 / 转客服）| 降级策略 |
| `embed(text)` | 任意文本 | 向量 | 语义检索、知识图谱 |
| `compare / guide / translate` | — | — | P1 预留 |

**合规红线（无论接哪家供应商）：**
- 用户账户数据、身份信息、交易数据永不出站
- AI 不给出投资建议、税务建议、个股推荐
- 税务/投资建议类问题统一回落人工话术
- 所有对话留存可审计；用户可清除；不进训练语料
- 引用的文档必须是"已审校"状态，过期文档自动剔除
- 每次调用记录 token 成本、延迟、拒答原因、用户反馈

**Provider 抽象层要求：**
- 输入侧：脱敏网关统一拦截（账号/身份证/金额/电话/邮箱），白名单字段才能进 LLM
- 输出侧：引用校验（模型输出的引用 id 必须在本次检索集合内，否则整条丢弃）
- 降级链：主 Provider 失败 → 备 Provider → 本地检索兜底（只返回段落，不生成答案）

## 7. 成功指标（北极星）

| 指标 | 定义 | 优先级 |
|---|---|---|
| **自助解决率** | 进入帮助后未在 N 分钟内提交工单的比例 | ★★★ |
| **AI 回答有用率** | 用户对 AI 回答标记"有帮助"的比率 | ★★★ |
| **任务完成率** | Playbook 中完成所有步骤的用户比例 | ★★★ |
| **客服工单减量** | 相同问题类型工单 MoM 下降 | ★★ |
| **App 内深链 CTR** | 帮助页跳回 App 完成操作的比率 | ★★ |
| **文档过期率** | `lastVerifiedAt` 过期的文档占比 | ★ |
| **AI 拒答合规率** | 超范围问题正确回落人工的比例 | ★★ |

## 9. 不做的事（YAGNI）

- 不做账户交易操作（不下单、不改密码、不入金）
- 不做社区 UGC（避免合规风险）
- 不做视频自制录制流程（先嵌入现有视频 CDN）
- 不做实时行情插件
- 不做 B 端理财经理工具（独立项目）

## 10. 开放决策（需对齐后再出技术方案）

1. **AI 供应商策略**：自研 RAG / Anthropic / OpenAI / 混合？（影响合规边界与成本）
2. **内容模型**：继续守 Markdown + frontmatter，还是引入 Headless CMS？
3. **登录态来源**：共享长桥主站 SSO，还是帮助中心独立轻登录？
4. **度量后端**：自建埋点，还是接公司现有 BI？
5. **Playbook 设计权**：运营/内容团队能否自己编排 Playbook？
6. **App "?" 入口**：App 团队是否全面替换为带 context 的帮助 URL？

# 文档优化任务清单

基于 Diataxis 框架对 `docs/` 目录（195 个文档）的分析，整理以下优化任务，按优先级逐一处理。

---

## 内容结构类

### T1 — 新增新手入门系列 `[Tutorial]`

**问题**：整个知识库没有任何教程型文档（Tutorial），新用户从开户完成到完成第一笔交易，没有引导路径。

**目标**：在 `docs/新手入门/` 下创建 3 篇教程：
- `完成首次入金.md` — 从选择入金方式到资金到账的完整操作过程
- `买入第一只港股.md` — 从搜索股票到下单成交的完整操作过程
- `设置股价提醒.md` — 设置提醒、接收推送的完整操作过程

**写作要求**（Diataxis Tutorial 原则）：
- 每一步都给出预期结果（"你会看到……"）
- 只走一条路，不提供多种选项
- 不解释"为什么"，只说"做什么"
- 结尾有明确完成状态

**状态**：`[ ] 待开始`

---

### T2 — 拆分 How-to 与 Reference `[Structure]`

**问题**：入金/出金类文档把"操作步骤"和"费率/规则"混写在一个文件里，用户找操作步骤时要跳过大量规则说明。

**涉及目录**：`docs/入金/`、`docs/出金/`

**目标**：每种入金方式拆成两个文件：
- `如何通过 FPS 转数快入金.md`（How-to：纯操作步骤）
- `FPS 转数快说明.md`（Reference：支持币种、到账时间、限额、收款信息、注意事项）

**示例**：`docs/入金/香港综合账户入金/FPS转数快.md` 拆分为上述两个文件。

**已完成**：入金目录 10 种方式 + 出金香港银行卡，共新增 11 个 How-to 文件，对应 Reference 文件清理为纯规则内容，双向交叉链接。

**状态**：`[x] 已完成（入金/出金）`

---

### T3 — 重构常见问题页面 `[FAQ Anti-pattern]`

**问题**：大量文档末尾附有"常见问题"章节，这些 Q&A 内容本质上是 How-to 或 Reference，堆在文章末尾导致内容重复、难以被检索。

**涉及文档（示例）**：
- `docs/衍生品/美股期权/期权基础.md` — 末尾 FAQ 应拆出
- `docs/融资/` 下各文档末尾 FAQ
- `docs/基金/` 下各文档末尾 FAQ

**目标**：
- 能单独成文的 FAQ 条目提取为独立 How-to 文档（如"如何提前平仓期权"）
- 纯规则说明的 FAQ 归入对应 Reference 文档
- 删除原文末尾重复的 FAQ 区块

**状态**：`[x] 已完成` — 涉及 19 个文件（4 个批次）的嵌入式 FAQ 全部移除，内容按 Diataxis 指南就地整合至主体对应章节；`营销活动与奖励/` 下的 Q&A 格式文档和独立 `*常见问题.md` 文件属于有意设计，保留不动。

---

### T4 — 扩充故障排查文档 `[How-to]`

**问题**：`docs/故障排查/` 目录几乎为空，用户遇到常见问题时找不到自助解决路径。

**目标**：新增以下故障排查文档：
- `入金未到账怎么办.md`
- `DDA 授权失败怎么办.md`
- `订单提交失败怎么办.md`
- `期权行权异常怎么办.md`
- `App 登录失败怎么办.md`

**写作要求**（Diataxis How-to 原则）：
- 面向已经遇到问题的用户，假设他们知道自己想做什么
- 用条件句处理分支（"如果显示 XXX 错误，则……"）
- 不解释背景知识，直接给出解决步骤

**状态**：`[x] 已完成` — 新增 5 个 How-to 故障排查文档，更新 index.md 分组导航。

---

### T5 — 重组产品功能目录 `[Reference]`

**问题**：`docs/产品功能/` 按 App 界面导航组织（客户端功能/行情/交易/资产/我的），而不是按用户业务需求组织，用户找不到"股票提醒"在哪个子目录下。

**目标**：将 `docs/产品功能/` 改为按业务域分类：
```
docs/产品功能/
  行情与数据/      # 股票详情、实时行情、基本面数据
  交易工具/        # 订单类型、条件单、止盈止损
  持仓与资产/      # 持仓详情、资产总览、损益分析
  提醒与通知/      # 价格提醒、消息推送
  账户设置/        # 安全设置、权限管理
```

**状态**：`[x] 已完成` — 去掉 `客户端功能/` 中间层，27 个文件全部上移，层级降至 3 层。

## 命名规范类

### T6 — 消除文件名与目录名重复

**问题**：存在 `融资常见问题/融资常见问题.md`、`港股打新/港股打新.md` 等目录名与文件名完全相同的情况。

**目标**：目录内唯一文件直接将内容上移，或将文件重命名为更具体的名称。

**示例修正**：
- `融资常见问题/融资常见问题.md` → `融资/常见问题.md` 或拆为多个具体文档

**状态**：`[x] 已完成` — T7 压缩层级后，dirname=filename 的冗余路径已自然消失。

---

### T7 — 压缩目录层级至最多 2 层

**问题**：部分路径达到 4-5 层（如 `docs/入金/香港综合账户入金/银行转账/汇丰银行.md`），导致 URL 过长、侧边栏层级过深。

**目标**：最大层级为 `docs/<分类>/<文档>.md`，最多 2 层（含 docs/）。

**具体调整**：用 Python 批量 git mv，149 个文件全部上移至顶层分类目录，并删除 51 个空目录。

**状态**：`[x] 已完成`

---

### T8 — 统一中文命名规范

**问题**：文件名中英文混用，如 `DDA.md`、`PayNow.md`、`FOK与IOC订单.md`、`FPS转数快.md`。

**目标**：
- 专有名词保留英文缩写（FPS、DDA、IPO），但配上中文说明
- 文件名统一：`FPS转数快.md` ✓，`PayNow即时转账.md` ✓，`DDA自动扣款授权.md` ✓
- 纯英文文件名改为中英结合：`FOK与IOC订单.md` → `FOK和IOC订单类型.md`

**已完成修改**：
- `DDA.md` → `DDA自动扣款授权.md`
- `PayNow.md` → `PayNow即时转账.md`
- `Wise.md` → `Wise国际汇款.md`
- `eDDA.md` → `eDDA电子授权扣账.md`
- `股票转仓HK.md` → `香港股票转仓.md`
- `股票转仓SG.md` → `新加坡股票转仓.md`

**状态**：`[x] 已完成`

---

### T9 — 文件名体现 Diataxis 文档类型

**问题**：文件名无法区分文档类型，`融资.md` 可能是操作指南也可能是概念说明。

**目标**：
- How-to 文档用动词开头：`如何通过X入金.md`、`如何出金到X.md`、`如何X.md`
- Reference 文档用名词：`X说明.md` 或直接名词形式 `X.md`
- Tutorial 文档（T1 新增）用"完成……"：`完成首次入金.md`

**已完成修改**：
- T2 入金/出金分拆时同步应用命名规则：所有操作步骤文件统一 `如何通过X入金.md` / `如何出金到X.md`
- `下载.md` → `如何下载长桥App.md`（纯 How-to 内容）

**未改（内容混合，待 T2/T3 后再命名）**：
- `登录注册.md`（包含 Reference 表格 + FAQ，先等内容拆分）
- `主子账户资金划转.md`（包含操作步骤 + 规则说明，先等内容拆分）

**状态**：`[x] 已完成（核心命名规则已确立，增量随 T2/T3 推进）`
